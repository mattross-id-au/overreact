Creating and nesting components

React apps Interactive websites are made out of components. A component is a piece of the UI (user interface) that has its own logic and appearance. A component can be as small as a button, or as large as an entire page.

Note on Shadow DOM

When rendering component child elements, the choice between attaching a Shadow DOM or using the Main DOM will mostly depend on the type of contents:

Attach a Shadow DOM
when the component is a self-contained widget not part of the main document, and child elements are for presenting the widget
Main DOM
When the component is generating child elements are to be part of the main document flow

Custom Element names must always contain a hyphen, must be lowercase and must always have both an opening and closing tag.

Writing markup with JSXtemplate literals

The markup syntax you’ve seen above is called JSXtemplate literals or template strings. It is optional, but most React projects use JSX for its convenience. All of the tools we recommend for local development support JSX out of the box.

JSX is stricter than HTML. You have to close tags like <br />. Your component also can’t return multiple JSX tags. You have to wrap them into a shared parent, like a <div>...</div> or an empty <>...</> wrapper:
