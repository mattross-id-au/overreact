Creating and nesting components

React apps Interactive websites are made out of components. A component is a piece of the UI (user interface) that has its own logic and appearance. A component can be as small as a button, or as large as an entire page.

Note on Shadow DOM

In the example above, the my-button element uses the Shadow DOM for it's child components as they make up self-contained functionality with encapsulated behaviour.

my-app on the other hand inserts page content for it's child elements, so we keep the generated elements in the main DOM.

Writing markup with JSXtemplate literals

The markup syntax you’ve seen above is called JSXtemplate literals or template strings. It is optional, but most React projects use JSX template literals for its convenience. All of the tools we recommend for local developmentmodern browsers support JSXtemplate literals out of the box.
